1.面向对象和⾯向过程的区别

  • 面向过程：性能高 （类的调用需要实例化，比较消耗资源）
  • 面向对象：易维护，复用，扩展 （封装，继承，多态的特性，可以实现低耦合）

2.Java特点

1.面向对象（封装，继承，多态）

2.平台无关性（JVM）

3.多线程

4.编译与解释并行

  • 编译是直接把高级语言编译成机器码

  • 解释不需要编译，在执行的时候进行翻译，一般而言效率比较低

  • .java -> 编译 -> .class ->交给JVM解释 比较特殊，需要编译而生成的.class 不是机器语言需要JVM来解释 因此 编译与解释并行

3.JVM、 JDK、 JRE的区别

1.JVM

JVM是运行Java字节码的虚拟机，不同系统使用相同的字节码，它们都会给出相同的结果。

字节码是.class文件，是JVM可以理解的代码。

2.JDK

即Java SDK 用于创建和编译程序

3.JRE

是Java运行时环境，但不能用于创建新程序

4.java和c++ 的区别

  • 都是面向对象的语言，都支持封装，继承和多态
  • java不提供指针来直接访问内存
  • java是单继承的，c++是多继承的 但java支持接口的多继承 也可以实现多个接口
  • java有自动内存管理机制
  • c语言中，字符串和字符数组最后会用一个字符’\0’ 来表示结束

5.字符型常量和字符串常量的区别

  • 位（bit） 数据存储的最小单位 1或0

  • 字节(byte) 8位

  • 字符 是单引号引起的一个字符 ‘a’ java使用的UTF-16编码 一个字符占两个字节

    char 16bit

    short 16bit

    int 32bit

    float 32bit

  • 字符串 是一对双引号括起来的多个字符的序列 “a”

6.重载和重写的区别

重载 overload

同一个类中 方法名相同 但是参数类型、个数、顺序等不同

重写 override

是子类对父类允许访问的方法进行重新编写
